Note: The reasoning behind having a max-size per crawler: Network calls will be the extremely dominant overhead. In order to
maximize network call throughput, we should partition our available RAM to a static number of crawlers. If each crawler maximally
uses the ram it's given most of the time (which should be the minimum amount of ram needed to not have a ridiculous number
of disk writes that would significantly slow down crawling), we will be maximizing network calls.
