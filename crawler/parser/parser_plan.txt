How crawl should work:


fd = <network call>
filename = place we wanna write this stuff
<class>buffer buf;
HTMLParser h;
while(ssize_t status = buf.read(fd) > 0):
    h.parse()
    buf.write(filename) // Write what we got then clear buffer to re use 
    addToURLStuff(h.urlStuff)

if status is -1 we die (not really we just don't do the write (or maybe we do cuz
we could write partially on a fail potentially))

if status is 0 (we finished read of all HTML):
    buf.write(filename) // Last write

Nice we read in HTML page parsed it then wrote it to file

if batch ready: send out urls to other machines' frontiers

Note: The reasoning behind having a max-size per crawler: Network calls will be the extremely dominant overhead. In order to
maximize network call throughput, we should partition our available RAM to a static number of crawlers. If each crawler maximally
uses the ram it's given most of the time (which should be the minimum amount of ram needed to not have a ridiculous number
of disk writes that would significantly slow down crawling), we will be maximizing network calls.
